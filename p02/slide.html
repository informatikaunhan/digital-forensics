‚ùì Mengapa Framework Penting?

Investigasi Forensik Digital adalah proses terstruktur untuk mengidentifikasi, mengumpulkan, memeriksa, menganalisis, dan melaporkan bukti digital yang relevan.
Aspek Tanpa Framework Dengan Framework
Proses Ad-hoc Sistematis

üèõÔ∏è NIST SP 800-86 Framework

Empat fase utama investigasi forensik digital:

1. Collection
Identifikasi & akuisisi data
2. Examination
Pemrosesan & ekstraksi data
3. Analysis
Interpretasi & korelasi temuan
4. Reporting
Dokumentasi & presentasi hasil

üí° Proses bersifat iteratif ‚Äî dapat kembali ke fase sebelumnya jika diperlukan

üì¶ Fase 1: Collection (Pengumpulan)

Aktivitas utama dalam fase pengumpulan:

  1. Identifikasi ‚Äî Menentukan perangkat & media yang relevan
  2. Prioritisasi ‚Äî Urutan berdasarkan volatilitas (Order of Volatility)
  3. Akuisisi ‚Äî Membuat salinan forensik
  4. Preservasi ‚Äî Menjaga integritas bukti
  5. Dokumentasi ‚Äî Mencatat setiap langkah
‚ö†Ô∏è Kesalahan di fase ini dapat menggagalkan seluruh investigasi!

üîç Fase 2 & 3: Examination & Analysis

Examination

  • Ekstraksi data dari image
  • Filtering data tidak relevan
  • Dekompresi & dekripsi
  • Kategorisasi artefak

Analysis

  • Timeline reconstruction
  • Correlation analysis
  • Pattern analysis
  • Attribution analysis

Bagian Konten

üö® First Response: Penanganan TKP Digital

First Response adalah serangkaian tindakan awal saat pertama kali tiba di lokasi insiden untuk mengamankan, mendokumentasikan, dan mempreservasi bukti digital.

First response yang tepat menentukan keberhasilan seluruh investigasi!

‚ùå Kesalahan di tahap ini ‚Üí bukti rusak, hilang, atau tidak admissible

üìã Langkah-Langkah First Response

  1. Perimeter Control ‚Äî Batasi akses ke area TKP
  2. Personnel Log ‚Äî Catat setiap orang masuk/keluar
  3. Threat Assessment ‚Äî Evaluasi risiko fisik & digital
  4. Authorization ‚Äî Pastikan otorisasi legal
  5. Documentation ‚Äî Dokumentasi kondisi awal
  6. Evidence Handling ‚Äî Penanganan sesuai status perangkat

üíª Penanganan: Live vs Dead System

üü¢ Sistem MENYALA

  1. JANGAN matikan langsung
  2. Foto/screenshot layar
  3. Catat waktu sistem
  4. Capture volatile data (RAM)
  5. Pertimbangkan live acquisition

üî¥ Sistem MATI

  1. JANGAN menyalakan!
  2. Foto semua sisi perangkat
  3. Cabut power dari belakang PC
  4. Label semua kabel & port
  5. Kemas dengan anti-static bag

‚ö†Ô∏è Mengapa Tidak Boleh Menyalakan?

Risiko menyalakan komputer yang sudah mati:

  • ‚è∞ Modifikasi timestamp ‚Äî OS mengubah metadata file saat boot
  • üíæ Overwrite data ‚Äî Startup menulis ke disk, menimpa bukti
  • ü¶† Autorun malware ‚Äî Script destruktif berjalan otomatis
  • üìã Registry changes ‚Äî Windows memodifikasi registry entries
  • üìä Log modification ‚Äî Event logs berubah dengan aktivitas baru

üîó Chain of Custody

Chain of Custody adalah dokumentasi kronologis yang mencatat setiap perpindahan, penanganan, dan penyimpanan bukti dari pengumpulan hingga pengadilan.

Jika chain of custody terputus, bukti dapat ditolak di pengadilan!

Prinsip Chain of Custody

  1. Continuity ‚Äî Tidak ada celah waktu dalam pencatatan
  2. Integrity ‚Äî Bukti tidak diubah, ditambah, atau dikurangi
  3. Accountability ‚Äî Setiap penanganan tercatat & bertanggung jawab
  4. Security ‚Äî Bukti di lokasi aman, akses terkontrol
  5. Documentation ‚Äî Setiap perpindahan didokumentasikan tertulis

Komponen Formulir Chain of Custody

Info Kasus

  • Nomor kasus
  • Tanggal/waktu
  • Investigator PJ
  • Unit/satuan

Deskripsi Bukti

  • Evidence number
  • Merek, model, S/N
  • Kondisi diterima
  • Hash value

Perpindahan

  • Dari siapa
  • Kepada siapa
  • Tujuan
  • Tanda tangan

üíø Forensic Imaging

Forensic Image adalah salinan bit-by-bit dari seluruh media, termasuk unallocated space, deleted files, dan metadata.
Jenis Cakupan Penggunaan
Physical Image Seluruh disk + unallocated Investigasi menyeluruh
Logical Image Hanya file/folder aktif Waktu terbatas
Targeted File tertentu saja Scope spesifik

üö´ Write Blocker

Write Blocker mencegah penulisan data ke media bukti, menjaga integritas bukti asli.

Hardware

  • Blok sinyal write secara fisik
  • Lebih reliabel
  • Tidak bergantung OS
  • Mahal, perlu dibawa fisik

Software

  • Modifikasi driver storage
  • Murah, mudah digunakan
  • Bergantung pada OS
  • Kurang reliabel

Algoritma hash untuk membuktikan integritas bukti:

üìë SOP Investigasi Forensik Militer

SOP adalah instruksi langkah demi langkah untuk melaksanakan prosedur secara konsisten dan sesuai standar.

Karakteristik SOP forensik militer:

  • Mengikuti hierarki komando militer
  • Mempertimbangkan klasifikasi keamanan
  • Selaras dengan doktrin operasi
  • Terintegrasi dengan sistem pelaporan militer
