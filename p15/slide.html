üîß Akuisisi Android via ADB

# Cek koneksi perangkat
adb devices

# Full backup
adb backup -all -shared -apk -system backup.ab

# Pull specific databases
adb pull /data/data/com.whatsapp/databases/msgstore.db

# Daftar semua packages terinstall
adb shell pm list packages

# Screenshot
adb exec-out screencap -p > screen.png

# Info perangkat
adb shell getprop ro.product.model
adb shell getprop ro.build.version.release

üìã Prosedur Akuisisi Android

  1. Persiapan: Aktifkan mode pesawat, cek baterai ‚â•50%
  2. Dokumentasi: Foto kondisi awal, catat model & IMEI
  3. USB Debugging: Settings ‚Üí Developer Options ‚Üí Enable
  4. Verifikasi: adb devices
  5. Backup: adb backup -all -shared
  6. Pull data spesifik: WhatsApp, DCIM, Download
  7. Hash: certutil -hashfile backup.ab SHA256
  8. Chain of custody: Dokumentasi lengkap

üóÑÔ∏è Analisis Database SQLite

Mayoritas data mobile tersimpan dalam SQLite:

-- Semua tabel dalam database
SELECT name FROM sqlite_master WHERE type='table';

-- Pesan SMS
SELECT address, body, date, type FROM sms 
ORDER BY date DESC;

-- WhatsApp messages
SELECT key_remote_jid, data, timestamp 
FROM messages ORDER BY timestamp DESC;

-- Call logs
SELECT number, duration, date, type 
FROM calls ORDER BY date DESC;

Tools: DB Browser for SQLite, sqlite3 CLI, Autopsy

üåê Bagian 2

Forensik Internet of Things (IoT)

Tantangan baru dalam investigasi forensik digital

üåê Apa itu Forensik IoT?

IoT Forensics adalah proses identifikasi, akuisisi, dan analisis bukti digital dari perangkat IoT yang terhubung ke internet.

Tantangan unik forensik IoT:

Heterogenitas
Beragam OS, protokol, format
Resource Terbatas
CPU, RAM, storage minimal
Volatilitas Tinggi
Data sering di-overwrite
Cloud Dependency
Data di server vendor

üèóÔ∏è Arsitektur IoT 3-Layer

APPLICATION LAYER
Dashboard, analitik, API ‚Üí Log aplikasi, user activity
NETWORK LAYER
Komunikasi, routing, gateway ‚Üí Traffic capture, logs
PERCEPTION LAYER
Sensor, aktuator, perangkat fisik ‚Üí Firmware, storage

Setiap layer menyimpan bukti forensik yang berbeda dan memerlukan pendekatan akuisisi yang unik.

üì° Protokol Komunikasi IoT

Protokol Transport Penggunaan Forensik
MQTT TCP Smart home, sensor Broker log analysis
CoAP UDP Constrained devices Packet capture
Zigbee 802.15.4 Home automation RF sniffing
BLE Bluetooth LE Wearables GATT analysis
Modbus TCP/Serial SCADA/ICS Register analysis

üîç Prosedur Forensik Drone

  1. Preservasi: Jangan nyalakan, lepas baterai, Faraday bag
  2. Dokumentasi: Foto semua sisi, catat serial number
  3. Akuisisi SD Card: Image menggunakan FTK Imager
  4. Ekstraksi Flight Log: DJI Assistant / DatCon
  5. Analisis Media: EXIF metadata dari foto/video
  6. Rekonstruksi Rute: Export KML ‚Üí Google Earth
  7. Identifikasi Operator: DJI account, paired devices

üìê Framework NIST SP 800-101

Panduan Forensik Perangkat Mobile:

1. Preservation ‚Üí Isolasi perangkat, Faraday bag, dokumentasi
2. Acquisition ‚Üí Pilih metode akuisisi yang tepat (L1-L5)
3. Examination ‚Üí Ekstraksi data menggunakan tools
4. Analysis ‚Üí Interpretasi dan korelasi data
5. Reporting ‚Üí Dokumentasi temuan dan kesimpulan

üéñÔ∏è Studi Kasus: Drone di Zona Militer

Drone tidak dikenal terdeteksi terbang di atas Kodam. Drone diamankan bersama smartphone diduga milik operator.
1. Preservasi
Lepas baterai drone, Faraday bag untuk smartphone
2. Akuisisi
Image SD card, ekstrak flight log, backup smartphone
3. Korelasi
Flight path ‚Üî location history, paired devices ‚Üî BT
4. Identifikasi
DJI account, foto aerial, timeline korelasi

üìã IoT Forensic Readiness (Militer)

  1. Inventarisasi: Katalog semua perangkat IoT dan klasifikasi criticality
  2. Logging: Implementasi logging di setiap layer (device, network, app, cloud)
  3. Retention: Kebijakan retensi log (90-365 hari)
  4. SOP Response: Prosedur dari detection ‚Üí reporting
  5. Training: Pelatihan forensik IoT untuk tim Cyber Defense
  6. Simulasi: Latihan insiden berkala

üìå Rangkuman

  • Mobile Forensics: Android (open, SQLite-based) vs iOS (encrypted, secure enclave)
  • Akuisisi: 5 level dari manual hingga chip-off, sesuaikan dengan kebutuhan
  • Artefak Kunci: SMS, WhatsApp, lokasi GPS, EXIF foto, browser history
  • IoT Forensics: Heterogen, resource terbatas, protokol khusus
  • Drone & SCADA: Kritis untuk keamanan militer, memerlukan tools khusus
  • Anti-Forensik: Factory reset, encryption ‚Üí counter dengan Faraday bag, cloud recovery
  • Framework: NIST SP 800-101 sebagai panduan investigasi mobile
