üåê Apa itu Forensik Jaringan?

Forensik Jaringan (Network Forensics) adalah cabang forensik digital yang berfokus pada pemantauan, penangkapan, pencatatan, dan analisis lalu lintas jaringan untuk tujuan pengumpulan bukti, deteksi intrusi, dan investigasi insiden keamanan.

Dua pendekatan utama (Marcus Ranum):

  • üìπ "Catch-it-as-you-can" ‚Äî Tangkap semua paket, analisis nanti
  • üëÅÔ∏è "Stop, look, and listen" ‚Äî Analisis real-time, simpan yang relevan

üìº Full Packet Capture (PCAP)

Gold standard forensik jaringan ‚Äî merekam keseluruhan paket termasuk header dan payload.

Informasi yang dapat diekstrak:

  • Konten komunikasi HTTP yang tidak terenkripsi
  • File yang ditransfer melalui jaringan
  • Kredensial yang dikirim dalam plain text
  • Query DNS dan respons
  • Metadata komunikasi terenkripsi (TLS handshake, certificate info)

‚ö†Ô∏è Estimasi Storage: Jaringan 100 Mbps, utilisasi 30% ‚Üí ~325 GB/hari. Gunakan retention policy!

üî¨ Analisis Protokol: TCP/IP

Setiap layer TCP/IP memberikan informasi forensik yang berbeda:

Layer Protokol Informasi Forensik
Application (7) HTTP, DNS, SMTP, FTP Konten, URL, email, file transfer
Transport (4) TCP, UDP Port number, session, flags
Internet (3) IP, ICMP IP sumber/tujuan, TTL, routing
Network Access (2) Ethernet, Wi-Fi MAC address, VLAN tags

ü§ù TCP Three-Way Handshake

Client (192.168.1.100)          Server (10.0.0.1:80)
         |                              |
         |---- SYN (seq=1000) -------->|    ‚Üê Inisiasi
         |                              |
         |<--- SYN-ACK (seq=2000, ------|    ‚Üê Konfirmasi
         |     ack=1001)                |
         |                              |
         |---- ACK (seq=1001, -------->|    ‚Üê Established
         |     ack=2001)                |

Implikasi Forensik: Banyak SYN tanpa ACK = SYN scan/SYN flood. SYN+SYN-ACK+RST = stealth port scanning.

üö© TCP Flags & Signifikansi Forensik

Flag Signifikansi Forensik
SYN Inisiasi koneksi; banyak SYN tanpa ACK = SYN scan/flood
ACK Koneksi established; ACK scan untuk bypass firewall
FIN Terminasi normal; FIN scan untuk stealth scanning
RST Koneksi direset; port tertutup atau filtered
PSH Push data; indikasi transfer data aktif 
# Wireshark: Deteksi SYN scan
tcp.flags.syn == 1 && tcp.flags.ack == 0

üåç Forensik DNS

DNS ‚Äî salah satu sumber bukti paling berharga karena hampir semua aktivitas internet dimulai dengan query DNS.
Elemen DNS Nilai Forensik
Query name Domain yang diakses oleh host
NXDOMAIN Domain tidak ada ‚Äî banyak NXDOMAIN = DGA malware
TXT records Sering digunakan untuk DNS tunneling
TTL rendah Indikator C2 fast-flux 
# Wireshark: DNS response NXDOMAIN
dns.flags.rcode == 3
# DNS TXT records (potential tunneling)
dns.qry.type == 16

üöá DNS Tunneling

DNS Tunneling ‚Äî menyalahgunakan protokol DNS untuk mengirim data melalui query/response, sering digunakan untuk data exfiltration karena DNS jarang diblokir. 
Query DNS Normal:
  www.example.com                        ‚Üí 15 karakter

Query DNS Tunneling:
  dGhpcyBpcyBhIHRlc3Q.data.evil.com     ‚Üí 34 karakter
  (Base64 "this is a test" sebagai subdomain)

Indikator: ‚úÖ Query name panjang (>30 char) ‚úÖ Volume tinggi ke satu domain ‚úÖ Tipe TXT/NULL ‚úÖ Entropy tinggi ‚úÖ Pola periodik

üîç Analisis DNS: Mana yang Mencurigakan?

Query 1: www.google.com                              (A record) ‚úÖ
Query 2: mail.kemhan.go.id                            (MX record) ‚úÖ
Query 3: dGhpcyBpcyBhIHRlc3Q.data.xyz123.net          (TXT)      ‚ùå
Query 4: c2VjcmV0IGZpbGUgY29udGVudA.data.xyz123.net   (TXT)      ‚ùå
Query 5: www.bing.com                                 (A record) ‚úÖ
Query 6: ZW5jcnlwdGVkIGRhdGEgaGVyZQ.data.xyz123.net   (TXT)      ‚ùå

Temuan: Query 3, 4, 6 = DNS Tunneling. Base64-encoded data dalam subdomain, semua menuju xyz123.net via TXT record. Decode: "this is a test", "secret file content", "encrypted data here".

üîÑ Metodologi Rekonstruksi Serangan

  1. Collection ‚Äî Kumpulkan semua PCAP, logs, flow data
  2. Filtering ‚Äî Filter noise; fokus IP/port/timeframe relevan
  3. Correlation ‚Äî Korelasikan event dari berbagai sumber
  4. Timeline ‚Äî Susun timeline kronologis
  5. Analysis ‚Äî Identifikasi teknik dan taktik penyerang
  6. Attribution ‚Äî Kaitkan dengan aktor ancaman yang dikenal
  7. Reporting ‚Äî Dokumentasi temuan dan rekomendasi

üïµÔ∏è Covert Channels

Covert Channel ‚Äî metode komunikasi menggunakan saluran/protokol yang tidak dimaksudkan untuk transfer data, untuk menyembunyikan komunikasi dari deteksi.
Metode Teknik Deteksi

üèì Contoh: Deteksi ICMP Tunneling

Temuan: Host 192.168.1.50 mengirim 200 ICMP echo request/menit ke 203.0.113.100, payload rata-rata 512 bytes.

Frekuensi 1-4/siklus 200/menit

Durasi Beberapa detik 24 jam terus-menerus

Estimasi Exfiltration: 512 bytes √ó 200/menit √ó 60 √ó 24 = ~140.6 MB dalam 24 jam!

üìå Key Takeaways

  1. Forensik jaringan fokus pada data in motion ‚Äî sangat volatile, perlu capture segera
  2. Tiga jenis bukti: Full PCAP (lengkap), Flow Data (efisien), Log Files (tersedia)
  3. DNS adalah goldmine ‚Äî hampir semua aktivitas dimulai dari DNS query
  4. Korelasi multi-sumber menghasilkan gambaran serangan yang lengkap
  5. Encrypted traffic tetap mengandung metadata yang berharga (SNI, JA3, certificate)
  6. Wireshark + Zeek + tcpdump = toolkit inti forensik jaringan
