üíæ Arsitektur Hard Disk Drive (HDD)

HDD adalah perangkat penyimpanan non-volatile yang menggunakan piringan magnetik berputar (platter) untuk membaca dan menulis data secara elektromagnetik.
Komponen Fungsi Relevansi Forensik
Platter Media magnetik penyimpanan Data recoverable setelah dihapus
R/W Head Membaca & menulis data Kerusakan ‚Üí data loss
Controller Mengontrol operasi disk Firmware menyimpan info forensik

üìê Geometri: CHS vs LBA

CHS (Cylinder-Head-Sector)
  • Cylinder: Track vertikal pada semua platter
  • Head: Sisi platter (0 = atas)
  • Sector: Subdivisi pada track (512 bytes)
  • Batas: ~8,4 GB
LBA (Logical Block Addressing)
  • Pengalamatan linier (mulai dari 0)
  • Lebih sederhana & modern
  • Mendukung kapasitas besar
  • Digunakan disk modern

‚ö° Solid State Drive (SSD)

SSD menggunakan chip memori flash NAND untuk menyimpan data secara elektronik ‚Äî tanpa komponen mekanik bergerak.
Aspek HDD SSD
Media Piringan magnetik Chip flash NAND
Komponen bergerak Ya Tidak
Recovery data Relatif mudah Sangat sulit
TRIM support Tidak Ya (menghapus data)

üî¨ Tipe Sel NAND Flash

Tipe Bit/Sel Daya Tahan Penggunaan
SLC 1 ~100K siklus Enterprise, militer
MLC 2 ~10K siklus Enterprise
TLC 3 ~3K siklus Consumer
QLC 4 ~1K siklus Storage massal

üí° SLC dipilih untuk militer karena keandalan tinggi dan rentang suhu operasi luas (-40¬∞C s/d 85¬∞C)

üìÄ Master Boot Record (MBR)

MBR adalah struktur 512 bytes di sektor pertama (LBA 0) yang berisi boot code dan tabel partisi. Dikembangkan IBM tahun 1983.
Offset Ukuran Komponen
0x000 446 bytes Bootstrap Code
0x1BE 4 √ó 16 bytes Partition Table (4 entri)
0x1FE 2 bytes Boot Signature (0x55AA)

üìä Struktur Partition Entry MBR

Setiap entri partisi = 16 bytes:

Offset Ukuran Field
0x00 1 byte Status (0x80 = bootable)
0x04 1 byte Partition type (0x07=NTFS, 0x0B=FAT32)
0x08 4 bytes LBA of first sector
0x0C 4 bytes Number of sectors
‚ö†Ô∏è Keterbatasan MBR: Maks 4 partisi primer | Maks 2 TB | Single point of failure

üõ°Ô∏è GUID Partition Table (GPT)

GPT adalah standar partisi modern (bagian spesifikasi UEFI), menggunakan GUID untuk identifikasi partisi.
Komponen Lokasi Deskripsi
Protective MBR LBA 0 Kompatibilitas mundur
GPT Header LBA 1 Metadata tabel partisi
Partition Entries LBA 2-33 Hingga 128 partisi
Backup Header + Entries LBA akhir Redundansi penuh

‚öñÔ∏è MBR vs GPT

Aspek MBR GPT
Kapasitas maks 2 TB 9,4 ZB
Partisi maks 4 primer 128
Redundansi Tidak ada Backup header & entries
CRC check Tidak Ya (CRC32)
Boot BIOS UEFI

FAT32 32 bit 2 TB 4 GB

üèóÔ∏è NTFS ‚Äî Sistem File Utama Windows

NTFS (New Technology File System) menyediakan journaling, MFT, ADS, enkripsi, dan metadata kaya ‚Äî sistem file paling sering ditemui dalam forensik.

üóÇÔ∏è Struktur Volume NTFS

üìä Master File Table (MFT)

Setiap file dan direktori memiliki minimal 1 entri MFT (biasanya 1024 bytes):

Offset Field Nilai Penting
0x00 Signature "FILE" (0x454C4946)
0x08 $LogFile Seq Number Nomor urut transaksi
0x16 Flags 0x01=In use, 0x02=Directory

üí° File yang dihapus: flag berubah dari 0x01 ‚Üí 0x00, tapi data MFT entry masih ada!

üóÉÔ∏è MFT System Files (Metafiles)

Record Nama Fungsi
0 $MFT Master File Table itu sendiri
1 $MFTMirr Mirror dari 4 record pertama
2 $LogFile Journal transaksi
5 . (root) Direktori root
6 $Bitmap Peta alokasi cluster
8 $BadClus Daftar cluster rusak

üè∑Ô∏è Atribut NTFS

Setiap MFT entry terdiri dari atribut-atribut:

Atribut Type Nilai Forensik

$FILE_NAME 0x30 Nama file, timestamps alternatif
$SECURITY_DESCRIPTOR 0x50 ACL dan permission
$DATA 0x80 Konten aktual file
$INDEX_ROOT 0x90 Struktur direktori (B-tree)

üïê NTFS Timestamps ‚Äî MACE

Timestamp $STANDARD_INFO $FILE_NAME
M ‚Äî Modified Terakhir konten diubah Diset saat file dibuat
A ‚Äî Accessed Terakhir diakses Diset saat file dibuat
C ‚Äî Created Waktu pembuatan Waktu pembuatan
E ‚Äî Entry Modified Terakhir MFT diubah Diset saat file dibuat
üö® Penting: $STANDARD_INFORMATION dapat dimanipulasi (timestomping), $FILE_NAME hanya diubah kernel ‚Äî lebih reliable!

üîç Deteksi Timestomping

Skenario: Analis BSSN menemukan anomali timestamp:

Atribut Created Dapat Dimanipulasi?
$STANDARD_INFO 2020-01-15 Ya ‚ö†Ô∏è

Kesimpulan: $SI Created lebih lama dari $FN Created ‚Äî secara logis tidak mungkin natural. Ini bukti timestomping (anti-forensik)!

üìì $LogFile vs $UsnJrnl

üí° Keduanya saling melengkapi: $LogFile untuk "bagaimana", $UsnJrnl untuk "siapa, kapan, apa"

üîí Alternate Data Streams (ADS)

ADS memungkinkan file NTFS memiliki beberapa stream data ‚Äî tidak terlihat melalui Windows Explorer atau dir standar.

Legitimate

  • Zone.Identifier (sumber download)
  • Thumbnail cache
  • Summary information

Malicious

  • Menyembunyikan malware
  • Exfiltration channel
  • Konfigurasi C2

üîé Membuat & Mendeteksi ADS

Membuat ADS:

echo "data rahasia" > file.txt:hidden_stream

Mendeteksi ADS:

dir /R
streams.exe file.txt    (Sysinternals)
üö® Kasus: File laporan.docx berukuran 15 KB di Explorer, tetapi FTK Imager menunjukkan 2,3 MB ‚Üí kemungkinan data rahasia disembunyikan dalam ADS!

üì¶ Cluster ‚Äî Unit Alokasi Terkecil

Cluster (allocation unit) adalah unit terkecil yang dialokasikan sistem file. 1 cluster = 1 atau lebih sektor berurutan.

2 ‚Äì 16 GB 4 KB 4 KB

üß© Slack Space

Slack Space adalah area tidak terpakai dalam cluster yang sudah dialokasikan ‚Äî berpotensi menyimpan data dari file sebelumnya.
RAM Slack
Antara akhir data file dan akhir sektor terakhir. Biasanya diisi nol (Windows modern).
Drive Slack
Antara akhir sektor terakhir file dan akhir cluster. Dapat berisi data lama!

üßÆ Perhitungan Slack Space

Contoh: File 5.000 bytes pada NTFS (cluster 4 KB):

Komponen Ukuran
Cluster dialokasikan ‚åà5.000/4.096‚åâ = 2 cluster
Ruang teralokasi 2 √ó 4.096 = 8.192 bytes
Slack space 8.192 ‚àí 5.000 = 3.192 bytes

üí° Slack space 3.192 bytes ini berpotensi menyimpan fragmen data dari file sebelumnya!

üìã Inode ‚Äî Metadata File Linux

Inode (Index Node) menyimpan metadata file pada ext ‚Äî setara MFT entry pada NTFS. Setiap file memiliki tepat 1 inode.
Field Deskripsi

üí° ext4 menyimpan deletion time (dtime) ‚Äî sangat berguna mengetahui kapan file dihapus!

üçé Sistem File Mac: HFS+ & APFS

üìì File System Journaling

Journaling mencatat perubahan sebelum diterapkan ‚Äî menjamin konsistensi setelah crash dan menyediakan "sejarah" untuk forensik.

‚ö° Forensik SSD ‚Äî TRIM

Tahap HDD SSD + TRIM

üîÑ Wear Leveling & Garbage Collection

Wear Leveling
  • Distribusi penulisan merata
  • Memperpanjang umur SSD
  • Data tersebar di lokasi berbeda
  • Recovery lebih sulit
Garbage Collection
  • Proses internal otomatis SSD
  • Menggabungkan halaman valid
  • Menghapus halaman invalid
  • Berjalan tanpa perintah user
